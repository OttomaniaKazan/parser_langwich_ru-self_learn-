# Парсер словаря с сайта langwitch.ru

> 🎓 Учебный проект по изучению веб-парсинга на Python

## 📋 О проекте

Этот проект создан в рамках обучения на курсе «Full-stack разработчик на Python» (Нетология). 

**Задача:** собрать базу английских слов с переводами для будущего телеграм-бота-тренажёра.  
**Решение:** написать парсер, который автоматически собирает слова с сайта `langwitch.ru` и сохраняет их в структурированном формате.

Проект стал первой практикой работы с:
- библиотекой `BeautifulSoup4` для парсинга HTML

---

## ✨ Возможности

- 📚 Автоматический сбор всех категорий слов с главной страницы `/wordsets/`
- 🔤 Извлечение пар «русское слово → английский перевод»
- 🧹 Очистка данных: удаление транскрипции, нормализация ключей (до 3 слов)
- 🔁 Глобальная дедупликация: исключение повторяющихся ключей между разделами
- 💾 Сохранение результата в читаемый JSON-файл с поддержкой кириллицы

## 📦 Формат выходных данных

Файл `vocabulary.json` содержит словарь, где:
- **Ключи первого уровня** — названия категорий (например, `"Овощи"`, `"Город"`)
- **Значения** — вложенные словари `{ "Русское": "English" }`

**Пример:**
```json
{
  "Овощи": {
    "Тыква": "Pumpkin",
    "Морковь": "Carrot",
    "Огурец": "Cucumber"
  },
  "Город": {
    "Улица": "Street",
    "Дом": "House",
    "Мост": "Bridge"
  }
}
```

**Особенности данных:**
- Русские слова приведены к формату с заглавной буквы
- Ключи нормализованы: оставлено не более 3 слов (например, «Кухня итальянская» → «Кухня итальянская», «Кухня, принадлежащая региону» → «Кухня принадлежащая»)
- Дубликаты ключей между категориями исключаются: при повторе сохраняется первое вхождение

## ⚙️ Как это работает (высокоуровнево)

```
1. main.py
   │
   ├─► sections_parse()
   │     └─► Загружает главную страницу, находит все категории
   │
   ├─► Цикл по категориям:
   │     │
   │     ├─► parse_section(category, seen_keys)
   │     │     ├─► Загружает страницу раздела
   │     │     ├─► Извлекает пары слов по классам .word_row_en / .word_row_ru
   │     │     ├─► Очищает английский текст от транскрипции
   │     │     ├─► Нормализует русский ключ (<=3 слова)
   │     │     ├─► Проверяет дубликаты через множество seen_keys
   │     │     └─► Возвращает словарь раздела + обновлённое seen_keys
   │     │
   │     └─► Добавляет результат в итоговый словарь
   │
   └─► save_to_json()
         └─► Сохраняет данные в data/vocabulary.json
```
